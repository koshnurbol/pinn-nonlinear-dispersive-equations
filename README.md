# Physics-Informed Neural Networks for Nonlinear Dispersive Equations

This repository contains the code accompanying the paper

> **Physics-Informed Neural Networks for Nonlinear Dispersive Equations**  
> N. Koshkarbayev, Institute of Mathematics and Mathematical Modeling, Almaty, Kazakhstan.

It is designed to reproduce the numerical results, ablation studies,
seed-robustness tables, diagnostic figures, and spectral-error analyses reported
in the paper for four canonical nonlinear dispersive equations:

| Equation | Operator | Domain | Final time |
|----------|----------|--------|------------|
| Korteweg--de Vries (KdV) | `u_xxx` | `[-10, 10]` | `1.5` |
| Benjamin--Bona--Mahony (BBM) | `u_xxt` | `[-20, 20]` | `5.0` |
| Kawahara | `u_xxxxx` | `[-20, 20]` | `2.0` |
| Rosenau--KdV | `u_xxxxt` | `[-20, 20]` | `2.0` |

All experiments use a standard fully connected physical-space PINN with `tanh`
activations and automatic differentiation for every derivative. No Fourier
features, spectral layers, or equation-specific basis functions are used.

running the scripts.

## Installation

```bash
python -m venv venv
source venv/bin/activate        # on Windows: venv\Scripts\activate
pip install -r requirements.txt
```

The code runs on CPU or GPU. A CUDA-capable GPU is recommended for the
fifth-order equations, especially Kawahara and Rosenau--KdV.

## Running the experiments

The KdV, BBM, and Kawahara scripts are controlled by three ablation flags near
the top of each file:

```python
SEED = 2026
USE_INPUT_NORMALIZATION = False
USE_ENERGY_LOSS = True
USE_LBFGS = True
```

The Rosenau--KdV script uses an `EXPERIMENTS` list containing the corresponding
ablation configurations. It also has a `FAST_MODE` flag for quick testing.

To run the main scripts from the repository root:

```bash
python src/pinn_kdv.py
python src/pinn_bbm.py
python src/pinn_kawahara.py
python src/pinn_rosenau.py
```

Each run prints the metrics, appends a row to a per-equation CSV file under

```text
results/raw/<equation>/
```

and saves generated figures under

```text
figures/generated/<equation>/
```

The reported metrics include final-time relative `L2` error, space-time `L2`
error, out-of-sample PDE residual, energy drift, and training time.

## Reproducing the ablation tables

The ablation tables in the paper correspond to toggling input normalization,
energy/invariant regularization, and Adam-to-L-BFGS refinement.

For example, for KdV:

| Case | `USE_INPUT_NORMALIZATION` | `USE_ENERGY_LOSS` | `USE_LBFGS` |
|------|---------------------------|-------------------|-------------|
| A0   | True  | True  | True  |
| A1   | False | True  | True  |
| A2   | True  | False | True  |
| A3   | True  | True  | False |
| A4   | False | False | True  |

The BBM, Kawahara, and Rosenau--KdV ablation tables follow the same pattern.
The configuration selected for each equation's seed-robustness study is:

| Equation | Selected config | Flags |
|----------|-----------------|-------|
| KdV          | A1 | no norm, energy, L-BFGS |
| BBM          | B2 | norm, no energy, L-BFGS |
| Kawahara     | K2 | norm, no energy, L-BFGS |
| Rosenau--KdV | R0 | norm, energy, L-BFGS |

## Reproducing the seed-robustness tables

Run the selected configuration with each of the three seeds used in the paper:

```python
SEED = 42      # then 123, then 2026
```

The reported values are the mean plus/minus sample standard deviation over the
three seeds.

## Reproducing the spectral-error analysis (Kawahara)

The Fourier error-spectrum figure and the phase-decomposition control are
produced from a trained Kawahara model. First train and save the model from the
repository root:

```bash
python src/pinn_kawahara.py
```

This writes the trained model to:

```text
models/kawahara_standard_pinn_model.pt
```

Then run the diagnostic scripts from the repository root:

```bash
python diagnostics/kawahara_error_spectrum.py
python diagnostics/kawahara_phase_decomposition.py
```

The diagnostic outputs are written to:

```text
results/raw/kawahara/
figures/generated/kawahara/
```

The first script reports the spectral centroids of the exact solution and of the
PINN error and saves the Hann-windowed Fourier error-spectrum figure. The second
diagnostic checks whether the high-wavenumber error content can be explained by
a rigid phase shift.

## Notes on reproducibility

- Random seeds are fixed for `torch` and `numpy`. Results on GPU may differ
  slightly from CPU results due to non-deterministic CUDA kernels; the reported
  trends and orders of magnitude are stable across hardware.
- The Rosenau--KdV script has a `FAST_MODE` flag. The default
  `FAST_MODE = False` reproduces the paper-scale results. Setting it to `True`
  runs a reduced-budget version for quick checks and will not match the paper
  numbers exactly.
- Generated CSV files, figures, and model checkpoints are intentionally ignored
  by Git and should be regenerated locally.
